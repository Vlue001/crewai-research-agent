# CrewAI Research Agent

> A multi-agent AI system that researches any topic, analyzes the findings, and writes a cited markdown report — built with [CrewAI](https://www.crewai.com/) and [Claude](https://www.anthropic.com/claude).

![Python](https://img.shields.io/badge/python-3.11+-blue.svg)
![License](https://img.shields.io/badge/license-MIT-green.svg)
![CrewAI](https://img.shields.io/badge/CrewAI-0.80.0-orange.svg)

Give it a topic and three specialized AI agents work in sequence — one searches the web, one extracts the key facts, and one writes a structured report — to produce a polished markdown document saved to disk.

## How it works

Three agents pass their work down an assembly line. Each is powered by Claude and has one job:

```
   "the Rust programming language"
                │
                ▼
   ┌────────────────────────┐
   │  1. RESEARCHER          │   Runs web searches (Serper.dev),
   │  Senior Research        │   gathers ~20 sources with URLs,
   │  Specialist             │   titles, and snippets.
   └───────────┬────────────┘
               │  raw findings
               ▼
   ┌────────────────────────┐
   │  2. ANALYST             │   Extracts the key facts, groups
   │  Data Analyst           │   them into themes, flags
   │                         │   contradictions, ranks sources.
   └───────────┬────────────┘
               │  structured analysis
               ▼
   ┌────────────────────────┐
   │  3. WRITER              │   Writes a markdown report with
   │  Technical Writer       │   summary, findings, analysis,
   │                         │   and numbered citations, then
   │                         │   saves it to outputs/.
   └───────────┬────────────┘
               │
               ▼
      outputs/{topic}_{timestamp}.md
```

CrewAI orchestrates the hand-offs (`Process.sequential`); each agent's output becomes the next agent's input.

## Features

- **3-agent sequential pipeline** — clean separation of research, analysis, and writing
- **Live web search** via the Serper.dev Google Search API
- **Cited markdown reports** with executive summary, key findings, analysis, and sources
- **Swappable model** — set `RESEARCH_MODEL` to use a cheaper model while developing
- **Fail-fast validation** — checks API keys before doing any work
- **Unit-tested tools** — both custom tools are covered by mocked `pytest` tests (no API calls needed)

## Quick start

### 1. Prerequisites
- Python 3.11 or newer
- An [Anthropic API key](https://console.anthropic.com)
- A [Serper.dev API key](https://serper.dev) (free tier: 2,500 searches)

### 2. Install
```bash
git clone https://github.com/Vlue001/crewai-research-agent.git
cd crewai-research-agent
pip install -r requirements.txt
```

### 3. Configure
```bash
cp .env.example .env
```
Then open `.env` and add your keys:
```
ANTHROPIC_API_KEY=sk-ant-...
SERPER_API_KEY=...
```

### 4. Run
```bash
python main.py "your research topic"
python main.py "quantum computing breakthroughs" --verbose
```
The finished report is saved to the `outputs/` directory.

## Choosing a model (cost control)

Every agent reads its model from `src/config.py`, which honors a `RESEARCH_MODEL` environment variable:

```bash
# Cheaper Haiku for development/testing
RESEARCH_MODEL="anthropic/claude-haiku-4-5" python main.py "test topic"

# Default (Sonnet 4.6) for final reports — just run normally
python main.py "real topic"
```

A typical run costs roughly **$0.10 on Haiku** or **$0.30 on Sonnet**. Serper search stays within the free tier.

## Project structure

```
crewai-research-agent/
├── src/
│   ├── agents/          # The three agent definitions
│   │   ├── researcher.py
│   │   ├── analyst.py
│   │   └── writer.py
│   ├── tools/           # Custom tools the agents use
│   │   ├── search_tool.py   # wraps the Serper.dev API
│   │   └── save_tool.py     # writes reports to disk
│   ├── tasks/
│   │   └── research_tasks.py # the three task definitions
│   ├── config.py        # single source of truth for the LLM
│   └── crew.py          # assembles agents + tasks into a Crew
├── tests/               # pytest tests for the custom tools
├── outputs/             # generated reports (git-ignored)
├── main.py              # CLI entry point
└── requirements.txt
```
